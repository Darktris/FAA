from sklearn.metrics import confusion_matrix
from sklearn.model_selection import ShuffleSplit

import itertools 

def plot_confusion_matrix(cm, classes,
                          normalize=False,
                          title='Confusion matrix',
                          cmap=plt.cm.Blues):
    """
    This function prints and plots the confusion matrix.
    Normalization can be applied by setting `normalize=True`.
    """
    if normalize:
        cm = cm.astype('float') / cm.sum(axis=1)[:, np.newaxis]
        print("Normalized confusion matrix")
    else:
        print('Confusion matrix, without normalization')

    print(cm)

    plt.imshow(cm, interpolation='nearest', cmap=cmap)
    plt.title(title)
    plt.colorbar()
    tick_marks = np.arange(len(classes))
    plt.xticks(tick_marks, classes, rotation=45)
    plt.yticks(tick_marks, classes)

    fmt = '.2f' if normalize else 'd'
    thresh = cm.max() / 2.
    for i, j in itertools.product(range(cm.shape[0]), range(cm.shape[1])):
        plt.text(j, i, format(cm[i, j], fmt),
                 horizontalalignment="center",
                 color="white" if cm[i, j] > thresh else "black")

    plt.tight_layout()
    plt.ylabel('True label')
    plt.xlabel('Predicted label')

Apartado 4 Comentarios sobre la validación cruzada¶

En s la validacin cruzada nos permite dividir un conjunto de datos en K particiones para la clasificacin, sin embargo, esto puede no ser tan estable segn el dataset.

El ejemplo ms claro de este fenmeno viene dado por el ordenamiento de los datos, esto lo que genera es que al estar ordenados los datos, al hacer la divisin en cada uno de los K-folds, resulte en que los datos de cada K-fold se parecen mucho entre s y en general poco a los de entrenamiento, lo que genera un error mucho ms alto del real. Para ello, se propone como solucin (implementada tambin en nuestras implementaciones), un shuffle previo de los indices antes de realizar la validacin cruzada o simple para as maximizar la distancia entre datos contiguos.

Por otro lado, la validacin cruzada devuelve K errores en vez de uno slo, lo que puede generar interpretaciones errneas de este valor ya que coger el mnimo o la media si hay mucha dispersin, puede resultar en dar una estimacin poco real del error.

Adems en el caso de modelos hiperparamtricos, donde previo a entrenamiento, se debe estimar un parmetro (el nico caso aqu es la correccin de Laplace y no se cumple esto), puede generar fallos ya que la estimacin y optimizacin del hiperparmetro estar realizada teniendo en cuenta todos los datos y no restringindose al conjunto de entrenamiento.

De manera emprica, se aade que el valor de la validacin cruzada viene dado solo por el obtener un valor de desviacin tpica del error, ya que la media del error es similar a la dada por validacin simple. Ntese que esto es algo particular del dataset y del clasificador, pero en este caso, no aporta un valor aadido.
